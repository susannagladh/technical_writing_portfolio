# Internal Technical Enablement Guide for Recruiters

## Overview

This project was an internal educational guide designed to help recruiters and sales consultants understand modern software development, technical roles, common terminology and the business value of technical competence.

The guide was created for a non-technical audience working with IT recruitment and consultant matching. Its purpose was to make complex technical topics easier to understand, discuss and apply in conversations with candidates, customers and internal teams.

The full document is not published due to confidentiality.

## Audience

The primary audience was recruiters and sales consultants who work with technical requirement profiles but do not necessarily have a background in software development.

The guide was written to support readers who need to understand technical roles, tools and concepts well enough to interpret customer needs, ask better questions and communicate more confidently about IT competence.

## Purpose

The purpose of the guide was to create a practical knowledge resource that could be used as:

- onboarding material for new employees
- a reference guide for technical terminology
- support when reading requirement profiles
- preparation before candidate or customer conversations
- internal training material about software roles and competence areas

## Content structure

The guide covers several areas of modern software development, including:

- basic software concepts
- the development process from requirements to delivery
- frontend, backend, APIs, fullstack and embedded systems
- cloud services and SaaS
- AI, machine learning and modern AI platforms
- programming languages, tools and databases
- common technical roles
- how technical competence creates business value
- a glossary of common technical terms

## Communication challenge

The main challenge was to explain technical concepts accurately without making the material too advanced for the intended audience.

The readers needed enough technical understanding to interpret requirement profiles and communicate with customers and candidates, but they did not need a developer-level explanation of each concept.

## Approach

I used a layered writing approach:

1. Start with simple explanations.
2. Add context and examples.
3. Connect each concept to real recruitment situations.
4. Summarize key points in recurring takeaway sections.
5. Link technical competence to business value.

To make the material easier to use, I included:

- short explanations
- practical examples
- comparison tables
- role summaries
- “common confusion” sections
- recruiter-focused takeaways
- a glossary for quick reference
